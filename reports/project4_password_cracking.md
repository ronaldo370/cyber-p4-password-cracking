## ­¤Äō Educational Concepts

### 1. Why are MD5 and SHA1 Insecure?
Algorithms like MD5 and SHA1 were designed for **high speed**. While speed is good for checksums, it is fatal for password security because an attacker can generate billions of hashes per second to brute-force combinations.

### 2. The Power of Dictionary Attacks
A dictionary attack is significantly more efficient than brute-force because it prioritizes words that humans are likely to choose (e.g., `letmein`, `password`). Using a large wordlist like `rockyou.txt` (containing 14 million passwords) allows attackers to compromise millions of accounts in minutes.

### 3. Defenses: Salting & Key Stretching
- **Salting**: Adding a unique random string to each password before hashing. This makes the hash unique even if two users have the same password, rendering precomputed tables (Rainbow Tables) useless.
- **Key Stretching (Argon2 / BCrypt)**: Modern algorithms are intentionally designed to be **computationally expensive** (slow). This makes it impractical for attackers to run large-scale dictionary attacks.

## ­¤øĀ’ĖÅ Remediation / Learnings

### Recommendations
1. **Enforce Strong Password Policies**: Minimum 12 characters, mixing uppercase, lowercase, numbers, and symbols.
2. **Deprecate MD5/SHA1**: Never use these algorithms for sensitive data storage.
3. **Use PBKDF2, BCrypt, or Argon2**: These are industry-standard "hard" hashing functions.

### Key Takeaways
- Even "standard" passwords like `admin` are completely insecure without proper hashing protections.
- Automated tools make it trivial to compromise weak security controls.
